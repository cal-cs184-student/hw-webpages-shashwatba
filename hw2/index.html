Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm is a recursive algorithm that allows us to construct Bezier curves using their control points. The algorithm works by recursively evaluating points somewhere between each pair of control points (weighted by parameter \(t\)) until we are left with only one point. The intermediate points calculated are the control points of the new Bezier curve. To implement it, I simply looped through all neighbouring points and evaluated their weighted average according to \(t\) and \((1-t)\) at each step of the proceses, storing them in a vector2D array of the original length minus one.

Part 2: Bezier surfaces with separable 1D de Casteljau

De Casteljau's algorithm extends to Bezier surfaces by first applying it to the rows of the control points. Each row produces a single final points, which is then used as the control points for a new Bezier curve. The algorithm is then applied to these remaining control points to produce a singular final point. To implement this, the core evaluateStep() function was really similar to part one. I then wrote the code to call evaluateStep() repeatedly until the size of the input points was one. Lastly, I wrote an algorithm to first evaluate the control points row-by-row, with u as the input parameter, and then evaluate the resulting points a single time with parameter v.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

To implement area-weighted vertex normals, I iterated through all the halfedges at the current vertex (until I looped around). Then, I computed the 3 verte positions corresponding to the face of the half-edge, and cross-producted their two differences. Then I added to the running sum return value the norm() of this cross times 0.5 times the norm. Lastly, I called .unit() to normalize the return vector.

Part 4: Edge flip

I found that the best way to implement edge flips was simply drawing out the two triangles before and after the edge split and assign edges, vertices, facese and halfedges to what they should be pointing to. The halfedges were the trickiest because half of them are on boundary edges, meaning the only way to get their corresponding facee is calling ->face() and I had to directly call ->next() to find out their invisible next edge. For debugging, I ended up just staring at my code really hard to ensure the cyclic patterns existed and made sense as per my drawing.

Part 5: Edge split

Implementing edge splits was similar to edge flips, except I had to be more careful. I drew out the diagrams before and after the split, and realized that I needed to create two new edges, along with their corresponding half-edges/twins, one vertex, and two new faces. I arbitrarily chose e0 to remain as the top-center edge in the diagram, along with some other arbitrary naming decisions. While creating new objects, I initialized them using the new[Name] functions, then calculated their attributes using my diagram. m's position was the midpoint of b and c's.

For debugging, I mostly found staring at my code helpful, and leaving and coming back to it in a while.
